# 游戏内战情简报指南

本指南供 skill 在信息不足、首次同步、排障和跨设备 handoff 时，把工具状态翻译成玩家能在 Civ6 里完成的操作。

## 面板位置

- 入口在 Civ6 原生左上 LaunchBar，按钮提示为「打开战情简报。」；对玩家可称为“左上副官入口”。
- 打开后面板标题是「战情简报」。
- 面板状态区显示两行：当前状态，以及「最近汇总：…」。
- 玩家点击汇总按钮后，成功状态应显示「简报已汇总，可继续由AI副官分析。」；最近汇总会显示「回合情报」「地图情报」「专题情报」「完整战情」或「自动汇总」以及回合数。
- 不要求玩家读取 `exportId`、chunk、checksum、manifest 或 sha256；这些只给日志、doctor、preflight 和 bridge 使用。

## 按钮语义

| 玩家按钮 | 当前导出内容 | 何时让玩家点击 |
|---|---|---|
| 「汇总本回合」 | 当前版本会刷新核心上下文：城市、单位、科技、市政、政体、政策、资源、公开外交和地图情报 | 首次使用、没有 snapshot、snapshot 过旧、用户不确定是否已同步、多个模块缺失、普通本回合优先级 |
| 「更新地图情报」 | `units`, `visibleMap`, `diplomacyPublic` | 开战、防守、侦察、海军、定居、前线、资源岛、地图为空或低置信度 |
| 「城市运营」 | `cities`, `resources` | 逐城生产、住房、宜居度、区域、城市节奏 |
| 「军事态势」 | `units` | 单位行动、调兵、防守、护送、可见敌军判断 |
| 「科技市政」 | `cities`, `techs`, `civics`, `resources` | 科技/市政路线、尤里卡、鼓舞、关键时机 |
| 「政体政策」 | `government`, `policies`, `resources` | 换卡、政策槽、当前政体、爆发回合 |
| 「资源库存」 | `resources` | 战略资源、奢侈品、升级、维护、交易和宜居度 |
| 「公开外交」 | `diplomacyPublic` | 已遇见文明、公开关系、公开军事分、战争风险 |
| 「完整战情简报」 | 所有主要模块和通知占位 | 版本变化、诊断异常、多个互不相关模块缺失、需要重建上下文 |
| 「回合开始自动汇总」 | 每个本地玩家回合开始时自动执行一次「汇总本回合」 | 用户希望减少每回合手动点击；默认关闭，仍需标准入口读取结果 |

## 给玩家的话术

无 snapshot 或首次使用：

```text
我还没读取到最新战情。请在 Civ6 点击左上副官入口，打开「战情简报」，点击「汇总本回合」。看到“简报已汇总，可继续由AI副官分析。”和“最近汇总：回合情报 …”后回到这里，我再读取最新战情继续分析。
```

地图/战争/定居信息不足：

```text
判断前需要前线可见地块和单位位置。请在 Civ6 左上副官入口打开「战情简报」，点击「更新地图情报」。看到“简报已汇总，可继续由AI副官分析。”后回到这里继续。
```

专题信息不足：

```text
当前快照缺少城市运营和资源库存，不能可靠给出逐城生产队列。请打开「战情简报」，选择「城市运营」和「资源库存」，看到简报已汇总后再继续。
```

多个模块缺失或状态混乱：

```text
当前战情上下文不完整。请打开「战情简报」，点击「完整战情简报」；如果只是想快速刷新本回合，也可以先点击「汇总本回合」。两者都只导出本地玩家理论可见信息。
```

## 平台回传

- 单机或同机分析：玩家完成点击后，终端按当前意图运行标准入口，例如 `npm run copilot -- --intent turn-priority --clean`。
- Windows 游戏机 + Mac Agent：Windows 侧完成点击后运行标准入口，生成并同步 handoff；Mac 侧读取 handoff。
- 如果用户说游戏界面由自己操作，AI 只给出战情简报按钮动作，并在终端侧运行标准入口或诊断命令。

## 排障优先级

- 面板打不开或左上没有副官入口：运行 `doctor`，检查 Mod 是否启用、是否进入真实对局、LaunchBar 按钮是否 attach。
- 面板显示成功但标准入口没有读取到新战情：运行 `doctor` 检查 Mod 日志、缓存状态和输出目录。
- `doctor` 缺少 `reason="exported"`：让玩家重新点击「汇总本回合」，并确认面板出现成功状态和最近汇总更新。
- manifest checksum/exportId 不一致：重新点击对应按钮，并重新运行标准入口。
- API 低置信度：让玩家重进真实对局后重新点击对应按钮；若仍低置信度，只能按低置信度来源分析。
