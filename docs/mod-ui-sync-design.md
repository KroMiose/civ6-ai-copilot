# 游戏内战情简报设计

战情简报是玩家在 Civ6 中汇总本地可见情报的入口。它应像游戏内战情面板，而不是开发者控制台：按钮清楚、状态简洁、技术细节交给日志和桌面工具。

## 1. 入口

- 入口挂在 Civ6 原生左上 LaunchBar 的 `ButtonStack`。
- 按钮使用 Civ6 可加载的原生图标/贴图作为主视觉信号。
- 加入按钮后 resize LaunchBar，避免静态坐标和原生 UI 重叠。
- 面板打开后贴近左上 LaunchBar 入口，避免悬浮在地图中央。
- 不使用强弹窗，不打断回合操作。

加载状态：

- XML context 先显示 `XML 已加载；等待 Lua 初始化。`。
- Lua 初始化成功后隐藏该诊断，并把面板切到 ready。
- 桌面工具结果由 doctor、manifest、preflight 和 handoff 显示，主面板只显示玩家可理解状态。

## 2. 布局

```text
战情简报

等待本回合情报汇总。
最近汇总：无

[汇总本回合] [更新地图情报]
[回合开始自动汇总：关闭]

专题情报
[城市运营] [军事态势] [资源库存]
[科技市政] [政体政策] [公开外交]

[完整战情简报]
```

按钮的适用场景和多人边界放入鼠标悬浮说明。面板正文只保留玩家当前需要判断的状态和动作。

## 3. 稳定按钮与模块

| UI 名称 | 模块键 | 场景 |
|---|---|---|
| 汇总本回合 | `turn` | 本回合基础上下文 |
| 更新地图情报 | `visibleMap` | 定居、战争、海军、前线、资源岛 |
| 城市运营 | `cities` | 逐城生产、区域、住房、产出 |
| 军事态势 | `units` | 侦察、防守、战争、单位行动 |
| 科技市政 | `techs`, `civics` | 尤里卡、鼓舞、路线规划 |
| 政体政策 | `government`, `policies` | 换卡、政策槽、爆发回合 |
| 资源库存 | `resources` | 战略资源、奢侈品、升级、交易 |
| 公开外交 | `diplomacyPublic` | 已遇见文明、公开关系、公开军事分 |
| 完整战情简报 | `full` | 首次使用、诊断、版本变化 |

这些稳定名称也是 skill 生成情报更新指令的来源。Skill 对玩家输出时按当前对话语言表达；需要精确点击当前简体中文界面时，引用这些按钮标签。改名必须同步更新 skill references、preflight、handoff 和测试。

## 4. 汇总类型

「汇总本回合」导出本回合综合分析所需的基础上下文。

「更新地图情报」导出本地玩家已揭示/当前可见地块，以及这些地块上的城市、资源、改良、地貌、己方单位和当前可见外方单位。它不是当前屏幕画面，也不是全图真相。

专题情报只汇总指定模块。未请求模块可以保留 schema 占位，但必须用 `modules` 和低置信度表达不可用于强结论。

「完整战情简报」用于首次使用、schema 变化、bridge 报错或多个关键模块需要更新。

## 5. 自动汇总

「回合开始自动汇总」默认关闭。开启后，Mod 只在本地玩家回合开始时调用现有「汇总本回合」导出路径，并按 `localPlayerId + gameTurn` 去重。

约束：

- 不新增采集路径。
- 不调用 `Request*`、`Set*` 或 gameplay 修改 API。
- 不改变 snapshot schema 主体。
- 触发状态通过面板与 `CIV6_AI_COPILOT_DIAGNOSTIC` 记录。

## 6. Skill 协作

Skill 判断信息不足时使用当前对话语言给出固定结构：先说明缺少什么，再给战情简报动作。当前简体中文界面的示例句式：

```text
请打开战情简报，点击「更新地图情报」。
```

```text
请打开战情简报，选择「城市运营」和「科技市政」。
```

面板只需要让玩家确认“我已经按要求汇总了情报”。桥接、checksum、manifest、schema 和 fairness 由桌面工具负责。

## 7. UI 文案边界

主面板可见文案使用简体中文。`bridge`、`preflight`、`exportId`、`sha256`、chunk、manifest 等技术词只出现在日志、manifest、doctor、preflight、handoff 和开发文档中。

按钮 tooltip 只说明按钮用途，不在游戏内重复多人公平声明。多人边界放在 README、玩家指南、skill 回复和桌面诊断中。
