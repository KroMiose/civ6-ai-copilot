# 语言与信息架构

本文定义 `civ6-ai-copilot` 的用户可见表达、文档分工和维护边界。它适用于游戏内文本、README、玩家指南、skill 回复、CLI 提示和发布材料。

## 产品气质

`civ6-ai-copilot` 是 Civilization VI 的 AI 回合副官。它应像游戏内战情参谋：克制、明确、可信，把当前局势整理成可行动的回合建议，同时尊重多人公平边界。

用户看到的文字应围绕三件事：

- 当前能确认什么。
- 下一步应该在游戏里做什么。
- 哪些判断受限于本地可见信息。

避免把产品写成开发工具、聊天机器人或调试面板。`snapshot`、`bridge`、`manifest`、`exportId`、`sha256`、`chunk` 等术语只出现在开发者文档、CLI、日志和诊断报告中；游戏内面板和普通玩家路径不解释这些实现细节。

## 用词原则

推荐用语：

- `战情简报`
- `汇总本回合`
- `更新地图情报`
- `专题情报`
- `本地玩家可见信息`
- `信息不足`
- `需要更新对应情报`
- `基于当前快照`
- `低置信度来源`

避免用语：

- 口语化的临时说明。
- 聊天机器人式的自我表态。
- 把游戏内功能写成开发工具或调试面板。
- 把实现细节写成普通玩家说明。
- 把具体 Agent 品牌写进游戏内界面。

## Skill 回复语言

`skill/SKILL.md` 主体使用简体中文维护，但运行时回复不强制中文。Skill 先根据用户最近一次明确提问语言确定当前对话语言；一旦确认，后续回答延续同一语言，除非用户明显切换语言或明确要求翻译。

回复不要给中英双语并列版本，也不要把术语写成中英混合解释。命令、路径、字段名、URL、内部标识符和未本地化专有名词可保留原文。游戏对象、地形、资源、区域、单位、科技、市政、政策和机制优先使用当前回答语言对应的 Civilization VI 本地化术语。

## 句式

信息足够时：

```text
基于当前战情简报，可以判断……
```

信息不足时：

```text
判断前需要前线可见地块和单位位置。请在战情简报中选择「更新地图情报」，待最新情报写入后再继续分析。
```

多人边界：

```text
以下建议仅基于本地玩家可见信息；不可见行动只作为风险推断。
```

## 文档分工

- `README.md`：简体中文玩家入口，包含项目定位、核心价值、安装入口和第一次使用提示。
- `README_en.md`：英文玩家入口，内容与 `README.md` 对齐，顶部与中文 README 互相切换。
- `docs/user-guide.md`：唯一的普通玩家安装、更新和日常使用指南。
- `docs/mod-installation-and-test.md`：开发者手动安装、诊断、测试和发布前验证。
- `docs/project-definition.md`：目标、非目标、关键决策、架构和契约。
- `docs/mod-ui-sync-design.md`：游戏内面板布局、按钮语义和 tooltip 规则。
- `docs/ai-analysis-view.md`：skill 如何使用快照、如何表达事实、风险和信息不足。
- `skill/SKILL.md`：运行时行为规则，避免复制完整用户教程。
- `skill/references/`：skill 可按需读取的安装、情报汇总、schema 和多人公平参考。

新增文档前先确认它是否承担新的信息所有权；如果只是重复已有步骤，应改为链接到权威文档。
